# day-reflection（日复盘）

## 定位

读取当天打卡 JSON，统计六类时间分配，对比人生目标，输出四层复盘报告。

**关键约束**：不修改原始数据；风格犀利直接，不给模糊安慰；每天只给一个行动建议。

---

## 输入

**数据来源**：`{workspace}/diary/YYYY/MM/YYYY-MM-DD.json`（路径从 MEMORY.md 读取）

```json
{
  "date": "2026-03-10",
  "timeline": [
    { "time": "09:00", "text": "起床", "supplement": false },
    { "time": "10:00", "text": "写代码", "supplement": false },
    { "time": "12:00", "text": "午休", "supplement": false },
    { "time": "23:00", "text": "刷手机", "supplement": false }
  ],
  "mood": "high",
  "insight": "今天效率不错",
  "tomorrow": "完成项目文档"
}
```

**时长计算**：相邻条目时差即为该条时长。最后一条无法计算时长时跳过。

**人生目标**：`{workspace}/diary/人生目标.md`（不存在时跳过 L3，其余三层正常输出）

---

## 六类时间定义（按关键词分类）

| 类别 | 关键词示例 |
|------|-----------|
| 🟢 核心工作 | 工作、开发、写代码、会议、项目 |
| 🔵 个人发展 | 学习、阅读、研究、练习 |
| 🟡 生活管理 | 吃饭、通勤、家务、起床、洗漱 |
| 🟠 社交娱乐 | 聊天、聚餐、游戏、电影 |
| ⚪ 休息 | 睡觉、午休、冥想 |
| 🔴 消耗/黑洞 | 刷手机、刷视频、发呆、摸鱼 |

无法匹配的归入「其他」，不强行分类。

---

## 输出（四层）

**输出位置**：`{workspace}/diary/YYYY/MM/YYYY-MM-DD_复盘.md`

```markdown
# YYYY-MM-DD 日复盘

## L1 时间统计

| 类别 | 时长 | 占比 |
|------|------|------|
| 🟢 核心工作 | 4h | 25% |
| 🔵 个人发展 | 1h | 6% |
| 🟡 生活管理 | 2h | 12% |
| 🟠 社交娱乐 | 0h | 0% |
| ⚪ 休息 | 7h | 44% |
| 🔴 消耗/黑洞 | 2h | 12% |

已追踪：16h

时间线：
09:00–10:00 起床·准备 (1h) 🟡
10:00–12:00 写代码 (2h) 🟢
...

## L2 模式洞察

**高效时段**：上午 10:00–12:00，连续专注 2h
**时间黑洞**：23:00 后刷手机 2h，占清醒时间 12%
**今日心情**：高效 🟢

## L3 身份检查

（对比 workspace/diary/人生目标.md）

✅ / ❌ 各角色今日投入 vs 目标
直接指出最大错位，一句话，不客气。

## L4 明日行动

一个可立即执行的小改变（具体到时间和动作）。
```

---

## 边界处理

| 场景 | 处理 |
|------|------|
| 记录 < 3 条 | 提示先用 diary-builder 完善，仍输出已有数据的统计 |
| 人生目标文件不存在 | 跳过 L3，提示"在 workspace/diary/ 创建人生目标.md 后可启用身份检查" |
| 今天是周日 | L4 末尾追加：「今天周日，可以回顾一下本周整体节奏。」 |
| 某类时间连续 14 天出现但不在目标中 | 在 L3 末尾建议新增角色，需用户确认后才写入目标文件 |

---

## 调用

```
/daily:reflect [YYYY-MM-DD]   # 日期可选，默认今天
```

**必须在 /daily:builder 之后运行**（或直接用网页提交的数据）。

---

## 反馈与迭代

每次运行结束后询问用户：
「这次的复盘有没有哪里不对劲？」

收到反馈后：
- 小调整（措辞/时间节点）→ 直接修改本 SKILL.md
- 功能新增 → 追加「待迭代」章节，用户确认后实施
- 记录到 MEMORY.md：日期 + 改了什么 + 为什么
