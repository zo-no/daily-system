# diary-builder（时间线补全）

## 定位

读取当天打卡数据，识别时间空白和模糊记录，通过追问补全，生成完整时间线。

**三条铁律**：
1. 原始记录一字不改
2. 补充内容必须标记 `[补充]`
3. 分类统计交给 day-reflection

---

## 网页版（自动）

`web/review.html` 已内置 diary-builder 逻辑，用户网页打卡时自动执行：
- 间隔 > 1h → 生成追问卡片
- 含糊动词（继续/玩/弄/逛…）→ 追问具体内容
- 深度工作关键词 → 追问产出

**网页版产出的数据已包含补充条目**（`supplement: true`），无需再次运行 diary-builder。

---

## AI 版（手动，用于深度追问）

当用户觉得网页版追问不够细，或想补充更多细节时，手动运行。

### 分析规则

| 类型 | 识别方式 | 追问内容 |
|------|---------|---------|
| 时段空白 | 相邻条目间隔 > 1h（排除睡眠时段） | 这段时间在做什么？ |
| 含糊动作 | 继续/玩/弄/逛/搞定/处理… | 具体是什么？ |
| 深度工作 | 写代码/学习/创作/研究… | 具体产出？遇到什么问题？ |
| 认知/感悟 | 突然的想法、洞察 | 是什么触发了这个想法？ |

**睡眠时段**：22:00–09:00 且间隔 > 6h，自动跳过不追问。

### 追问格式

批量编号，选项 + 自由填写，用户可用 `1.A 2.自定义内容` 快速回答：

```
【时段空白】
1. 13:00–18:00（5h）这段时间在做什么？
   A. 休息/午睡  B. 外出  C. 其他：____

【信息模糊】
2. 10:06 "出发继续玩" — 具体去哪里？
   A. 景点游览  B. 购物  C. 其他：____
```

### 输出格式

```
[补充] 07:00 起床
10:06 出发继续玩。
[补充] 10:06–11:00 从酒店步行至钟楼
11:00 到达钟楼
```

---

## 文件路径

- **读取**：`{workspace}/diary/YYYY/MM/YYYY-MM-DD.json`（只读，路径从 MEMORY.md 获取）
- **写入**：`{workspace}/diary/YYYY/MM/YYYY-MM-DD_AI生成.md`（新文件，不覆盖原 JSON）

---

## 边界处理

| 场景 | 处理 |
|------|------|
| 没有记录 | 提示先填写网页打卡 |
| 只有 1 条记录 | 以该记录为锚点，向前向后询问 |
| 用户说"记不清了" | 记为 `[补充] XX:XX 记忆模糊时段`，不强迫回忆 |

---

## 调用

```
/daily:builder [YYYY-MM-DD]   # 日期可选，默认今天
```

运行后建议接着运行 `/daily:reflect` 做时间统计分析。

---

## 反馈与迭代

每次运行结束后询问用户：
「这次的整理有没有哪里不对劲？」

收到反馈后：
- 小调整（措辞/时间节点）→ 直接修改本 SKILL.md
- 功能新增 → 追加「待迭代」章节，用户确认后实施
- 记录到 MEMORY.md：日期 + 改了什么 + 为什么
