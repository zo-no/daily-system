# daily-morning（早报）

## 职责

在早晨触发，读取昨日复盘，结合人生目标，生成今日待办推荐。

**不是任务管理器**，不追踪完成情况。只是每天早晨的一次「方向确认」。

---

## 触发时间

`config.morning_time`（默认 08:00，首次部署时由用户配置）。

---

## 输入

昨日日期记为 `YESTERDAY`（格式 `YYYY-MM-DD`），路径从 MEMORY.md 读取。
YYYY/MM 取自 YESTERDAY 日期本身（例如昨天是 2026-03-31，则目录为 2026/03）。

1. **昨日复盘**：`{workspace}/diary/YESTERDAY的YYYY/YESTERDAY的MM/YESTERDAY_复盘.md`
2. **人生目标**：`{workspace}/diary/人生目标.md`
3. **昨日日记原始数据**：`{workspace}/diary/YESTERDAY的YYYY/YESTERDAY的MM/YESTERDAY.json`（提取 `tomorrow` 字段）

---

## 输出格式

```
早上好。

昨天 [一句话概括昨日状态，基于复盘的 L2 模式洞察]。

今天建议：
• [待办 1]（来源：昨日未完成 / 人生目标错位 / 用户自填 tomorrow）
• [待办 2]
• [待办 3，最多 3 条]

今日基调：[一句话，基于昨日心情和目标错位判断]
```

### 示例

```
早上好。

昨天核心工作只有 2h，下午大块时间被刷手机吃掉了。

今天建议：
• 上午 10:00 前开始写项目文档（你昨天说今天要完成）
• 减少手机使用，试试把手机放到另一个房间
• 留 30 分钟读书（个人发展已连续 3 天为 0）

今日基调：专注一件事，别让手机偷走上午。
```

---

## 边界处理

| 场景 | 处理 |
|------|------|
| 昨天没有写日记 | 早报照常发，开头改为「昨天没有记录。」，待办仅基于人生目标生成 |
| 人生目标文件不存在 | 仅基于昨日 `tomorrow` 字段生成待办，跳过目标对照 |
| 连续 3 天没有日记 | 在早报末尾加：「最近几天没有记录，要不要今晚试着写一下？」 |
| 今天是周一 | 在末尾加：「新的一周，可以考虑更新一下人生目标。」 |

---

## 人生目标动态更新

daily-morning 不直接修改人生目标，但会在发现以下情况时提出建议：

- 某类时间连续 7 天出现但不在目标中 → 建议新增角色
- 某个角色连续 14 天投入为 0 → 建议确认是否仍有效

**用户确认后，由 agent 写入 `{workspace}/diary/人生目标.md`**，不自动修改。

---

## 调用

```
/daily:morning [YYYY-MM-DD]   # 日期可选，默认今天（生成昨日复盘的早报）
```

---

## 反馈与迭代

每次运行结束后询问用户：
「这次的早报有没有哪里不对劲？」

收到反馈后：
- 小调整（措辞/待办数量）→ 直接修改本 SKILL.md
- 功能新增 → 追加「待迭代」章节，用户确认后实施
- 记录到 MEMORY.md：日期 + 改了什么 + 为什么
