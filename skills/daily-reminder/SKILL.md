# daily-reminder（行为规范提醒）

## 定位

根据每日/每周复盘分析，自动识别需要规范的行为，创建定时提醒，帮助人类养成好习惯。

## 核心功能

1. **分析复盘** - 读取复盘报告，识别需要改进的行为
2. **设置提醒** - 根据分析结果设置定时提醒
3. **执行提醒** - 到时间自动执行提醒（say 命令）
4. **跟踪效果** - 记录提醒执行情况

## 工作流程

```
1. 触发：每日复盘后（或手动）
2. 分析：读取复盘报告，提取"明日行动"项
3. 判断：哪些需要设置定时提醒
4. 设置：写入 reminder 配置
5. 执行：到时间自动 say 提醒
```

## 分析规则

根据复盘 L3/L4 内容识别需要提醒的行为：

| 行为类型 | 关键词 | 提醒示例 |
|----------|--------|----------|
| 睡眠 | "早点睡"、"23:00"、"睡眠不足" | "该睡觉了" |
| 学习 | "学习时间"、"输入不够"、"阅读" | "今天学习了吗" |
| 运动 | "运动"、"锻炼"、"身体" | "该运动了" |
| 休息 | "休息"、"放松"、"别太累" | "休息一下吧" |
| 工作 | "超时"、"太晚"、"通宵" | "该休息了" |

## 提醒设置

### 配置格式

写入 `{workspace}/diary/reminders.json`：

```json
{
  "reminders": [
    {
      "id": "sleep_001",
      "type": "sleep",
      "time": "23:30",
      "message": "指挥官，该睡觉了！",
      "enabled": true,
      "created": "2026-03-10",
      "reason": "今日复盘：凌晨2点才睡，需要规范睡眠"
    }
  ]
}
```

### 执行方式

通过 `say` 命令（在 Mac 上语音提醒）：

```bash
say "指挥官，该睡觉了！"
```

或在指定时间通过 cron/schedule 执行。

## 调用方式

```
/daily:reminder           # 分析今日复盘，设置提醒
/daily:reminder list     # 列出当前所有提醒
/daily:reminder delete [id] # 删除提醒
/daily:reminder enable/disable [id] # 启用/禁用提醒
```

## 输出

1. **成功设置**：告知设置了什么提醒
2. **无需设置**：告知为什么不需要新提醒
3. **执行提醒**：到时间自动 say 提醒

## 示例

**输入**：复盘报告显示"明日行动：23:30 前睡觉"

**输出**：
```
✅ 已设置睡眠提醒
- 时间：23:30
- 内容："指挥官，该睡觉了！"
- 原因：今日复盘显示凌晨2点才睡
```
