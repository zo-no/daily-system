# 升级说明

> 面向 agent。用户说「更新日记系统」时，执行 `git pull` 后，**必须读本文件**，按当前版本对应的章节执行迁移操作。

---

## 如何使用本文件

1. 执行 `git pull`
2. 读取新版本号（`plugin.json` 的 `version` 字段）
3. 找到对应章节，执行迁移操作
4. 告知用户新增了什么

---

## 1.2.0（当前版本）

**新增功能**：网页打卡 Step 1 支持粘贴流水账

**迁移操作**：无需迁移，纯前端新增，不影响已有数据和配置。

**告知用户**：
```
日记系统已更新到 1.2.0 ✓

新功能：打卡网页现在支持直接粘贴流水账。
打开链接后，点「粘贴流水账」，把今天的记录贴进去，自动解析为时间条目。
```

---

## 1.1.0

**变更**：移除白天打卡，改为仅晚间一次；新增早报 Skill；所有时间点改为用户可配置。

**迁移操作**：

1. 检查 `{workspace}/config.json` 是否存在以下字段，不存在则补充：
   ```json
   {
     "morning_time": "08:00",
     "poll_interval_minutes": 30,
     "give_up_time": "01:00"
   }
   ```
2. 检查 `{workspace}/HEARTBEAT.md`，确认包含早报定时任务：
   ```
   - {morning_time} /daily:morning 早报
   ```
   不存在则追加。

**告知用户**：
```
日记系统已更新到 1.1.0 ✓

主要变化：
• 取消了白天打卡，只保留晚间一次记录
• 新增早报：每天早上会收到今日建议
• 所有时间点现在可以在配置里自定义
```
